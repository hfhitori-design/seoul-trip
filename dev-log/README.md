# 开发日志

## 规范

- 每个开发日一个文件，命名格式 `YYYY-MM-DD.md`
- 由 Claude 在每个开发会话中维护
- 记录两个部分：**完成事项** 和 **待办事项**

## 文件命名

```
dev-log/
├── README.md          # 本文件
├── 2026-06-27.md      # 示例
└── YYYY-MM-DD.md      # 后续日志
```

## 约定

1. 每次开发前，Claude 应读取最新日期的日志了解当前状态
2. 每次开发后，Claude 应更新当日日志（新建或追加）
3. 待办事项中未完成的项目，后续日志中持续跟踪
